# Kanban 看板项目

现代化任务管理看板应用，支持响应式设计和 GitHub Pages 部署。

## 功能特性

### 核心功能
- ✅ **添加任务**: 响应式模态框，支持输入验证，Session Storage 存储
- ✅ **任务管理**: 待办/进行中/已完成三列看板，支持拖拽
- ✅ **任务筛选**: 按优先级筛选（高/中/低）
- ✅ **实时搜索**: 任务标题、ID、负责人实时搜索（300ms 防抖）
- ✅ **任务详情**: 点击查看详情，支持 Markdown 描述渲染
- ✅ **统计面板**: 总任务数、总工时、完成率（可拖动位置）

### 数据分析与可视化
- ✅ **人员负荷热力图**: ECharts 热力图，展示 8 名团队成员在 7 天迭代周期内的工时负荷分布
  - 颜色编码：🟢 正常(0-6h) 🟡 饱和(6-8h) 🔴 过载(>8h) ⚪ 无任务
  - 交互功能：悬停提示、点击查看任务详情
- ✅ **数据分析页面**: 综合数据可视化面板
  - 📊 统计面板：4 个关键指标卡片（任务总数、完成率、平均工时、团队成员）
  - 🥧 角色分布饼图：各角色工时占比（策划/服务端/客户端/测试）
  - 📊 工时类型堆叠柱状图：任务工时 vs 非任务工时对比

### 存储方案
- **Session Storage**: 数据在会话期间有效，刷新页面不丢失
- **无服务器存储**: 适合 GitHub Pages 静态部署

### 兼容性
- ✅ iOS Safari 完美支持
- ✅ 触摸事件优化
- ✅ 响应式设计（桌面/平板/手机）
- ✅ 弹性滚动支持

## 技术栈

- **原生 HTML/CSS/JavaScript**: 无框架依赖，轻量级
- **ECharts 5.x**: 数据可视化图表库
- **Session Storage**: 现代浏览器存储 API
- **响应式设计**: CSS Grid + Flexbox
- **iOS Safari 优化**: 触摸事件和弹性滚动

## 部署方案

### GitHub Pages 部署

**准备工作**:
1. 创建 GitHub 仓库
2. 配置 GitHub Pages 设置
3. 部署项目文件

**部署脚本**:
```bash
# 部署到 GitHub Pages
git add .
git commit -m "feat: 完成添加任务功能和响应式优化"
git push origin main
```

### 本地开发

```bash
# 启动本地服务器
python3 -m http.server 8889
# 访问 http://localhost:8889/kanban.html
```
