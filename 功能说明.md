# 添加任务功能说明

## 功能特点

### 🎯 核心功能
- ✅ **本地显示**：添加的任务立即在看板显示
- ✅ **实时更新**：看板实时刷新，无需手动操作
- ✅ **纯内存运行**：不保存到任何持久化存储
- ✅ **iOS Safari 兼容**：完美支持移动端浏览器

### ⚠️ 重要说明
**数据不会保存！**
- 添加的任务仅在当前浏览器会话中显示
- 刷新页面后，任务会消失
- 关闭浏览器后，任务会消失
- 不会修改服务器上的任何数据

## 使用方法

### 添加任务
1. 点击右上角 **"➕ 添加任务"** 按钮
2. 填写任务信息：
   - **任务ID**：唯一标识（如 XY-026）
   - **任务标题**：任务名称
   - **状态**：待办/进行中/已完成
   - **优先级**：高/中/低
   - **参与人员**：可添加多个
   - **审核人员**：可添加多个
3. 点击 **"添加任务"** 按钮
4. 看板自动刷新，显示新任务

### 查看结果
- 新任务会立即出现在对应状态的列中
- 显示绿色提示：**"任务添加成功！（仅在当前会话显示）"**
- 可以继续添加更多任务

### 数据生命周期
```
添加任务 → 内存显示 → 关闭/刷新页面 → 数据消失
```

### 关键代码

#### 添加任务（纯内存）
```javascript
const newTask = {
    task_id: taskId,
    title: taskTitle,
    status: taskStatus,
    priority: taskPriority,
    iteration_name: iterationName,
    participants: participants,
    reviewers: reviewers
};

// 只添加到内存，不保存
state.allTasks.push(newTask);
state.filteredTasks = [...state.allTasks];

// 立即显示
renderBoard();
closeModal();
showSuccessMessage('任务添加成功！（仅在当前会话显示）');
```

#### 加载数据（仅从服务器）
```javascript
async function loadTasks() {
    const response = await fetch('data/projects/xiyouji/tasks.json');
    const data = await response.json();
    state.allTasks = data.tasks || [];
    state.filteredTasks = [...state.allTasks];
    // 不从 localStorage 加载
}
```

## 常见问题

### Q1: 为什么刷新后任务消失了？
**A**: 这是设计行为。任务只保存在浏览器内存中，刷新页面会重新从服务器加载原始数据。

### Q2: 能否保存到本地文件？
**A**: 不能。浏览器安全限制，前端无法直接修改本地文件。需要后端服务支持。

### Q3: 添加的任务会影响服务器数据吗？
**A**: 不会。添加的任务仅在浏览器内存中，不会发送到服务器，也不会修改任何服务器文件。

### Q4: 如何让任务持久化？
**A**: 需要开发后端功能：
1. 搭建后端服务器（Node.js/Python/Java等）
2. 实现数据库存储（MySQL/MongoDB等）
3. 提供 REST API
4. 修改前端代码，调用 API 保存数据

### Q5: 能否导出任务数据？
**A**: 当前版本不支持。可以考虑添加导出功能（JSON/CSV格式）。
