### 表单字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务标题 | 文本 | ✅ | 5-100字符，支持 emoji |
| 优先级 | 选择 | ✅ | 高/中/低（默认中） |
| 负责人 | 选择 | ✅ | 从项目人员池选择 |
| 预估工时 | 数字 | ✅ | >=1小时，整数 |
| 任务描述 | 多行文本 | ❌ | 可选，支持 Markdown |
| 验收人 | 选择 | ❌ | 可选，默认与负责人相同 |

### 交互流程
1. 点击「添加任务」按钮（右上角或列底部）
2. 响应式模态框显示表单
3. 实时字段验证（输入时）
4. 提交按钮 loading 状态
5. 数据保存后自动关闭表单
6. 看板刷新，新任务显示在「待办」列

- [ ] 模态框响应式布局

## MVP 10: 数据持久化 (0.5天) 📋

### 目标
实现数据的本地持久化。

### 验收标准
- [ ] LocalStorage 数据存储
- [ ] 页面刷新后数据不丢失
- [ ] 数据导出功能（JSON 下载）
- [ ] 数据导入功能（JSON 上传）
- [ ] （可选）带过期时间的存储封装

### 技术说明
```javascript
// 基础使用
localStorage.setItem('kanban_tasks', JSON.stringify(tasks));
const tasks = JSON.parse(localStorage.getItem('kanban_tasks')) || [];

// 带过期时间的封装（类似 Cookie）
const Storage = {
    set: (key, value, expireSeconds) => {
        const data = {
            value: value,
            expire: expireSeconds ? Date.now() + expireSeconds * 1000 : null
        };
        localStorage.setItem(key, JSON.stringify(data));
    },
    get: (key) => {
        const data = localStorage.getItem(key);
        if (!data) return null;
        const parsed = JSON.parse(data);
        if (parsed.expire && Date.now() > parsed.expire) {
            localStorage.removeItem(key);
            return null;
        }
        return parsed.value;
    }
};
```
