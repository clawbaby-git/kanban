# 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
面向敏捷开发团队的项目管理可视化工具，帮助管理者监控和管理每周一次的迭代进度。

### 1.2 参考项目
本项目参考 [Plane](https://github.com/makeplane/plane) 的设计理念，Plane 是一个开源的 Jira/Linear/Monday/ClickUp 替代品，主要功能包括：

- **Work Items（工作项）** - 任务创建与管理，支持富文本编辑和文件上传
- **Cycles（周期/迭代）** - 团队冲刺管理，包含燃尽图等进度追踪工具
- **Modules（模块）** - 将复杂项目分解为可管理的子模块
- **Docs（文档）** - 项目文档管理
- **Roadmaps（路线图）** - 产品路线图规划

本项目的差异化定位：
- **轻量级** - 纯前端实现，无需后端服务
- **数据驱动** - 以 JSON 数据为核心，便于与其他系统集成
- **可视化优先** - 专注于数据可视化展示和决策支持
- **本地化** - 支持离线使用，数据存储在本地

### 1.3 项目目标
- 输入: 结构化数据文本（JSON）
- 输出: 网页可视化仪表盘
- 核心价值: 实时监控迭代进度、资源分配优化、团队协作效率提升

## 2. 数据模型定义

### 2.1 项目配置 (config.json)
- 项目名称、迭代信息、日期范围
- 角色定义、人员资源池

### 2.2 任务数据 (tasks.json)
- 任务基础信息（ID、标题、状态、优先级）
- 参与者（人员、角色、预估/实际工时）
- 验收人（人员、预估/实际工时）

### 2.3 非任务工时 (non-tasks.json)
- 类型：会议、学习、其他
- 日期、人员、角色、内容、工时

## 3. 功能需求

### 3.1 核心看板功能
- 三列看板（待办/进行中/已完成）
- 卡片拖拽移动
- 添加/编辑/删除任务

### 3.2 可视化仪表盘
- 迭代燃尽图
- 人员负荷热力图
- 资源结构分析
- 工时类型分布

## 4. 非功能需求

### 4.1 性能要求
- 首屏加载 < 3秒
- 交互响应 < 100ms
- 支持 100+ 任务卡片流畅渲染

### 4.2 兼容性
- 现代浏览器（Chrome/Firefox/Safari/Edge 最新2版本）
- 响应式设计，支持平板和桌面
